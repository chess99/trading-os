# Trading System — AI 交易系统编排

这是整个交易系统的入口和调度中心。它的职责是理解用户意图，选择合适的子流程，
并把各个专项 skill 的输出串联起来，形成完整的交易决策链。

---

## 系统架构

```
trading-system（本 skill，编排层）
├── signal-scanner    扫描候选标的，生成信号清单
├── elder-screen      单标的三重滤网深度分析
├── position-sizer    仓位计算（2%/6%原则）
├── trade-executor    生成具体交易指令
├── position-monitor  持仓监控与止损追踪
├── trading-journal   交易记录与绩效统计
└── backtest-review   系统回测与健康评估
```

所有子 skill 均位于同一 `skills/` 目录下。

## 流程模式

根据用户的请求，选择对应的流程模式：

### 模式 A：完整日常流程（最常用）

适用于每个交易日开始时，从头到尾跑一遍。

```
1. [signal-scanner]  扫描候选标的池 → 输出信号清单
2. [elder-screen]    对信号清单中的高优标的逐一深度分析
3. [position-sizer]  对确认的交易机会计算仓位
4. [trade-executor]  生成今日交易指令清单
5. [position-monitor] 检查现有持仓状态，更新止损线
```

### 模式 B：信号扫描

用户只想看今天有什么机会，不需要立刻执行。

```
1. [signal-scanner]  扫描候选标的池
2. [elder-screen]    对 TOP 信号做简要分析（可选）
```

### 模式 C：持仓管理

用户已有持仓，想检查状态。

```
1. [position-monitor] 检查所有持仓的止损状态
2. [trade-executor]   如有需要，生成调仓指令
```

### 模式 D：交易记录与复盘

```
1. [trading-journal]  记录新完成的交易
2. [backtest-review]  定期（每周/每月）评估系统表现
```

## 编排原则

**顺序依赖**：信号 → 分析 → 仓位 → 指令，这个顺序不能颠倒。
没有信号就没有分析，没有分析就不能计算仓位，没有仓位就不能生成指令。

**风险门控**：在进入 trade-executor 之前，必须确认：
- 信号已通过三重滤网（第一滤网和第二滤网方向一致）
- 仓位已按 2% 原则计算
- 本月已用风险额度 + 本次风险 ≤ 账户的 6%

任何一项不满足，停在当前步骤，不进入下一步。

**透明输出**：每个步骤完成后，向用户汇报结果摘要，等待确认再继续。
不要在用户不知情的情况下自动进入下一步（尤其是 trade-executor）。

## 会话状态

在一次完整的交易流程中，维护以下状态，供各步骤使用：

```
账户信息:
  - 账户总资金
  - 本月已亏损金额
  - 现有持仓列表（标的、入场价、止损价、仓位）

今日分析结果:
  - 信号清单（来自 signal-scanner）
  - 深度分析结果（来自 elder-screen）
  - 待执行指令（来自 position-sizer + trade-executor）
```

---

## 启动方式

用户首次使用时，询问：
1. 账户规模（用于 2%/6% 计算）
2. 候选标的池（用于 signal-scanner 扫描）
3. 现有持仓情况（如有）

这些信息可以保存在会话中，下次使用时直接确认是否有变化。
