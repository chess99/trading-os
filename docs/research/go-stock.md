# go-stock 项目深度调研报告

> 仓库：https://github.com/arvinlovegood/go-stock
> 调研日期：2026-04-06

## 1. 定位与目标用户

- **核心定位**：基于 LLM 的 AI 赋能股票分析工具（桌面应用）
- **目标用户**：散户/个人投资者为主
- **人的角色**：半自动化决策辅助——AI 提供选股建议，最终决策权在人；无真实下单能力

## 2. 系统架构

**技术栈**：
- 前端：Vue 3 + NaiveUI + Vite
- 后端：Go + Wails 2（桌面应用框架）
- 数据库：SQLite（WAL 模式）
- AI：Cloudwego eino（字节开源 Agent 框架）+ OpenAI 兼容接口
- 爬虫：Chromedp（Go Headless Chrome）+ 浏览器连接池

**核心目录**：
```
backend/
├── agent/          # React Agent + 16个专用工具
├── data/           # 60+ 数据服务（东财爬虫、Tushare、情感分析）
├── db/             # SQLite GORM（WAL+连接池）
└── models/         # 90+ 数据结构体
```

**设计思路**：Wails 单文件可执行 + 本地 SQLite + 可插拔 LLM + 浏览器池爬虫

## 3. AI/Agent 使用方式

**框架**：Cloudwego eino 的 React Agent

```go
agent, _ := react.NewAgent(*ctx, &react.AgentConfig{
    ToolCallingModel: model,
    ToolsConfig:      tools,  // 16个工具
    MaxStep:          20,
})
```

**支持模型**：OpenAI、火山方舟（DeepSeek/字节）、Ollama 本地模型

**决策链路**：
```
用户输入 → React Agent → Tool Calling（选股/K线/新闻/情感分析）→ LLM 推理 → 流式输出
```

**亮点**：
- 支持思考模式（Extended Reasoning）
- 多轮对话持久化到 SQLite
- 多提示词模板（激进型/价值型/技术型视角可切换）

## 4. 数据来源与管理

| 数据类型 | 来源 | 更新频率 |
|---------|------|--------|
| A 股 K 线 | 东财爬虫（ChromeDP） | 实时/5秒 |
| 港美股 K 线 | Tushare | 日级 |
| 市场资讯 | 财联社、新浪财经爬虫 | 实时流 |
| 研究报告 | 机构网站爬虫 | 定时 |
| 股票基础数据 | Tushare JSON（编译嵌入） | 启动初始化 |

**存储**：SQLite 本地，WAL 模式，连接池 max 5

**支持市场**：A 股（全面）、港股/美股（基础，有延迟）

## 5. 策略层

**支持策略**：
- 技术面：MACD 金叉、KDJ 金叉、均线多头排列、放量突破
- 基本面：自然语言驱动（PE/PB/ROE 筛选）
- 情感面：中文金融词典打分（100+正面词+100+负面词，带权重）

**回测**：❌ 无回测系统，只有手动交易记录

**信号生成**：定时 Cron 任务触发 AI 分析 + 实时资讯 AI 总结

## 6. 执行层

- **模式**：纯模拟，无真实下单接口
- **账户管理**：SQLite 本地交易记录（止损/止盈配置、浮动盈亏计算）
- **通知**：钉钉推送提醒用户手动下单
- **风控**：涨跌幅预警 + 止损止盈记录，无自动执行

## 7. 亮点与可借鉴设计

1. **eino React Agent + Tool Calling**：避免手写 Agent 循环，天生支持并行工具调用
2. **提示词模板系统**：提示词存数据库，用户可切换分析视角
3. **浏览器连接池**：复用 Chrome 连接，爬虫速度提升 3-5 倍
4. **中文金融情感词典**：程度副词乘数 + 否定词反转，领域专用
5. **流式输出**：实时反馈，避免 UI 卡顿
6. **定时任务 UI 配置**：Cron 表达式可在界面配置，无需改代码

## 8. 局限性与应避免的设计

| 缺陷 | 具体问题 | 建议 |
|------|---------|------|
| 无回测 | 无法验证策略有效性 | 必须有回测引擎 |
| 无实盘接口 | AI 信号无法自动执行 | 集成券商 API |
| 数据源脆弱 | 严重依赖东财爬虫，反爬随时失效 | 多源冗余（Akshare/baostock） |
| 港美股延迟 | Tushare 延迟 15-30 分钟 | 实时数据源 |
| 无投资组合管理 | 只记录单股，无整体风险视图 | Portfolio 聚合层 |
| 定时任务不持久 | 进程内存储，软件关闭任务丢失 | 持久化到数据库 |
| AI 决策黑盒 | 推荐原因不透明 | 决策链路日志 + 得分分解 |

## 9. 对本系统的启示

**值得采用**：
- Tool Calling Agent 范式（比直接 Prompt 更稳定可控）
- 提示词模板系统（配置化，不硬编码）
- 中文金融情感分析思路

**需要超越**：
- 架构：桌面单机 → Web 服务分离
- 数据：单源爬虫 → 多源冗余
- 执行：纯记录 → 实盘接口
- 风控：警报级 → 硬性风控（VaR、压力测试、熔断）
- 回测：无 → 完整回测引擎
