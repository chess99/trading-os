## Problem Statement

已有基于 Elder《以交易为生》的技术分析 skills（8 个）。
需要决定：是否加入其他流派的 skills？加哪些？如何组织目录和使用流程？Agent 如何决定用哪种方法？

---

## 核心洞察

不同流派的分析维度是**正交的，不是竞争的**：

- **基本面**：这家公司值不值钱（长期价值锚点）
- **技术面**：市场现在是否认可这个价值（入场时机）
- **资金/情绪面**：短期动力够不够（催化剂）

三个维度都绿灯时信号最强。这不是"选一种流派"的问题，而是"哪个维度在当前场景最重要"的问题。

**最实用的组合**（机构实践验证）：基本面选股 + 技术面确认买点。
欧奈尔的 CANSLIM 是这个组合最系统化的实现。

---

## 决策：加什么，不加什么

### 加入

**威廉·欧奈尔《笑傲股市》→ `canslim-screen` skill**

理由：
- CANSLIM 是目前最成熟的"基本面选股 + 技术面确认"结合框架
- 和 elder-screen 有明确分工：CANSLIM 回答"值不值得关注"，elder-screen 回答"现在能不能进"
- 可操作性强，有具体的量化筛选条件（EPS 增长 > 25%、ROE > 17%、机构持仓增加等）
- A 股适配性好（成长股逻辑在 A 股同样有效）

CANSLIM 七个维度：
- **C** — 当季 EPS 增长 > 25%
- **A** — 年度 EPS 连续增长 3 年以上
- **N** — 新产品/新管理层/新高（突破前期高点）
- **S** — 流通股少 + 机构持仓增加
- **L** — 行业龙头（相对强度排名前 10%）
- **I** — 机构资金流入
- **M** — 大盘方向（只在牛市做多）

### 暂不加入

| 书/大师 | 原因 |
|---------|------|
| 彼得·林奇《成功投资》 | 框架和 CANSLIM 高度重叠，欧奈尔更系统化 |
| 达利欧《债务危机》 | 宏观框架难以转化成每日操作 skill |
| 施瓦格《市场奇才》 | 原则集合，不是操作框架 |
| Van Tharp《通向金融自由之路》 | 仓位管理增强，可以后期集成进 position-sizer，不需要单独 skill |

```
.claude/skills/
├── 编排层
│   └── trading-system/         ← 入口，识别意图，调度子流程
```

## Agent 的流程决策逻辑

Agent 不自由选择流派，由 `trading-system` 里的**预定义流程**处理。

### 流程 A：基本面优先（适合长线选股）

触发词："这只股票值得长期持有吗"、"帮我筛选成长股"、"基本面怎么样"

```
1. canslim-screen    → 基本面评分（C/A/N/S/L/I/M 各维度打分）
2. elder-screen      → 技术面确认（现在是好的入场时机吗）
3. position-sizer    → 仓位计算
4. trade-executor    → 生成指令
```

### 流程 B：技术面优先（适合短线机会）

触发词："扫描今天的机会"、"有什么突破信号"、"这个位置能进吗"

```
1. signal-scanner    → 技术面批量扫描
2. elder-screen      → 三重滤网深度分析
3. canslim-screen    → 基本面验证（这不是在炒梦吧）
4. position-sizer    → 仓位计算
5. trade-executor    → 生成指令
```

### 流程 C：综合分析（默认）

触发词："分析一下 600000"、"这只股票能买吗"

```
1. canslim-screen    → 基本面评分
2. elder-screen      → 技术面分析
3. position-sizer    → 仓位计算（两个维度都通过才计算）
```

**关键规则**：两个维度冲突时（基本面好但技术面差，或反过来），明确告诉用户冲突点，不强行给出买卖建议。

## canslim-screen Skill 设计要点

### 数据来源

A 股基本面数据从 trading-os 获取：
```bash
# 财务数据（EPS、ROE、营收）
python -m trading_os fetch-ak --exchange SSE --ticker 600000 ...

# 或者直接查询 AKShare 基本面接口
# akshare.stock_financial_report_sina()
# akshare.stock_roe_em()
```

### 输出格式

```
CANSLIM 评分：SSE:600000 贵州茅台
═══════════════════════════════════
C — 当季EPS增长：+32%  ✓（>25%）
A — 年度EPS增长：连续5年  ✓
N — 近期新高：距历史高点 -8%  ⚠（未突破）
S — 流通股：适中，机构持仓增加  ✓
L — 行业地位：白酒龙头 #1  ✓
I — 机构资金：近3月净流入  ✓
M — 大盘方向：震荡，谨慎  ⚠

总评：5/7 维度通过
建议：基本面优质，等待技术面突破信号再入场
→ 下一步：运行 elder-screen 确认入场时机
```

## 成功标准

- 用户说"分析 600000"，agent 自动先跑 CANSLIM 再跑三重滤网，输出两个维度的综合结论
- 两个维度冲突时，agent 明确说明冲突，不强行给建议
- 目录结构清晰，新加的 skill 能被 trading-system 正确调度

## What I noticed about how you think

- 你说"说实话我不知道"然后马上想出了一个有实践价值的答案（基本面选股 + 技术分析确认买点）。这不是不知道，这是在实时推理。
- 你问"这样是否合理"——但你已经描述了机构投资者实际在做的事情。直觉比你以为的更准。
- 你倾向于问"哪种最好"，但你真正想要的是"哪种组合在什么场景下最有效"。这是更成熟的问题框架。
