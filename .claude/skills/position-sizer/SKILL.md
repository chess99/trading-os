# Position Sizer — 仓位计算器

埃尔德的核心洞见：大多数交易者毁于仓位过重，而不是方向判断错误。
2%原则保护你免于单笔灾难，6%原则保护你免于连续亏损的雪崩效应。

---

## 核心规则

### 2% 原则（单笔上限）

每笔交易承担的最大风险 = 账户总资金 × 2%

```
最大仓位 = (账户总资金 × 2%) ÷ (入场价 - 止损价)
```

**示例：**
- 账户：100,000 元
- 单笔最大风险：2,000 元
- 入场价：50 元，止损价：48 元（每股风险 2 元）
- 最大仓位：2,000 ÷ 2 = **1,000 股**

### 6% 原则（月度上限）

本月已亏损 + 所有持仓的风险敞口 ≤ 账户月初资金 × 6%

当两者之和达到 6% 时，本月停止开新仓。

---

## 计算流程

### 第一步：收集必要信息

需要以下输入：
```
账户信息:
  - 账户总资金（月初值）
  - 本月已实现亏损金额

现有持仓风险（每个持仓）:
  - 入场价
  - 当前止损价
  - 持仓数量
  → 持仓风险 = (入场价 - 止损价) × 数量
  → 注：如果止损已移至盈亏平衡点以上，该持仓风险为0

本次交易:
  - 标的名称
  - 入场价（计划）
  - 止损价（计划）
  - 交易品种（股票/期货/外汇）
```

如果用户没有提供，逐项询问。

### 第二步：计算本次最大仓位

```python
# 基础计算
max_risk_per_trade = account_capital * 0.02
risk_per_unit = entry_price - stop_price
max_position = max_risk_per_trade / risk_per_unit

# 期货/外汇需要考虑合约乘数
# 股票直接用股数
```

### 第三步：检查6%原则

```python
monthly_loss = 本月已亏损
existing_risk = sum(每个持仓的风险敞口)
new_trade_risk = risk_per_unit * max_position

total_risk = monthly_loss + existing_risk + new_trade_risk
risk_limit = account_capital * 0.06

if total_risk > risk_limit:
    # 需要减少仓位或放弃交易
    available_risk = risk_limit - monthly_loss - existing_risk
    if available_risk <= 0:
        # 本月不能开新仓
    else:
        # 仓位需要缩减
        adjusted_position = available_risk / risk_per_unit
```

### 第四步：止损价位建议

- 止损价应设在支撑位/阻力位附近
- **避开整数价格**（改为整数上下几个最小单位）
- 止损距离至少为 1 倍 ATR，避免被市场噪音触发
- 止损不能太紧（会被震出）也不能太宽（风险过大）

## 输出格式

```
# 仓位计算结果

## 账户状态
- 账户总资金：100,000 元
- 本月已亏损：1,200 元（1.2%）
- 现有持仓风险：3,500 元（3.5%）
- 已用风险额度：4,700 元（4.7%）
- 剩余可用额度：1,300 元（1.3%）← 距6%上限

## 本次交易
- 标的：AAPL
- 入场价：182.50
- 止损价：179.75（建议，支撑位180下方）
- 每股风险：2.75 元

## 计算结果
- 2%原则上限：2,000 元 → 最多 727 股
- 6%原则约束：剩余额度 1,300 元 → 最多 472 股
- **建议仓位：472 股**（受6%原则约束）
- 实际风险：1,299 元（1.3%）
- 本次交易后累计风险：6,000 元（6.0%）← 达到月度上限

⚠️ 注意：执行此交易后，本月不能再开新仓（已达6%上限）

## 风险收益评估
- 止损距离：2.75 元（1.5%）
- 目标1（1倍ATR）：185.30 → 盈利 2.80 元/股
- 目标2（2倍ATR）：188.10 → 盈利 5.60 元/股
- 风险收益比：1.0:1 / 2.0:1
```

## 特殊情况处理

**如果6%额度已满：**
```
⛔ 本月可用风险额度已耗尽（已用 X%，上限6%）
本月剩余交易日内，不建议开新仓。
可以：
1. 管理现有持仓（调整止损、部分止盈）
2. 等待下个月重置
3. 如现有持仓止损上移至盈亏平衡，会释放部分额度
```

**如果止损太近（< 1倍ATR）：**
```
⚠️ 止损距离（X元）小于1倍ATR（Y元）
过紧的止损容易被市场噪音触发，建议将止损移至 Z 元
调整后仓位：N 股
```

**如果风险收益比不足2:1：**
```
⚠️ 当前风险收益比为 1.2:1，低于建议的2:1
建议：
1. 寻找更好的入场点（降低入场价）
2. 调整目标价（是否有更大的上涨空间？）
3. 或者跳过这笔交易，等待更好的机会
```

## Trading OS 集成

仓位计算结果直接对应 `Signal.size` 字段（占 NAV 的比例）：

```python
# 2% 原则计算结果示例
# 账户 100万，止损 5%，则单笔风险 = 100万 × 2% = 2万
# 仓位 = 2万 / (入场价 × 5%) = X 股
# 对应 Signal.size = X股 × 入场价 / 100万 NAV

Signal("SSE:600000", "BUY", size=0.08)  # 8% NAV
```

RiskManager 会在执行前再次校验：单股上限 10%，月度总风险 6%。
如果 position-sizer 算出的 size > 10%，RiskManager 会拒绝信号。
两层保护：skill 层计算 + 代码层强制执行。
