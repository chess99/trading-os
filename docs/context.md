# 背景与工作约定（新会话先读）

## 我们要做什么
目标是搭建一套“以赚钱为最终目的”的个人交易系统（Trading OS），覆盖：
- 数据（行情/财务/事件/情绪等）
- 筛选（screeners）
- 研究与回测（验证想法）
- 执行与风控（纸交易→实盘）
- 决策记录与复盘（形成可检索、可迭代的知识库）

## 当前关键选择（已达成共识）
- 市场范围：**多市场（A股/美股/港股）**，但避免一开始就被接口与规则差异拖复杂。
- 落地路径：**先纸交易闭环**（研究→回测→执行→风控→记录→复盘），稳定后再接入真实券商接口做实盘。

## 设计原则（避免“回测幻觉”）
- **口径优先**：任何指标/因子/收益都要明确数据口径、复权、时区、交易日历。
- **可复现**：研究结论应能被 Notebook/脚本一键复现。
- **评估真实**：费用、滑点、成交规则、停牌/涨跌停（A股）等要可插拔，但不能长期缺失。
- **风控先行**：在实盘前，风控与记录要比策略更“硬”。
- **记录驱动迭代**：每次下单/信号都要能追溯“当时为什么”，复盘能形成具体改进动作。

## 仓库怎么用（约定）
- `docs/`：系统设计与方法论（对齐认知、减少口头沟通成本）
- `journal/`：你个人的决策/复盘（Markdown，按模板写，方便检索）
- `notebooks/`：可复现研究（尽量少“手工点点点”）
- `data/`、`artifacts/`：本地数据与产物 **不入库**（已在 `.gitignore`）
