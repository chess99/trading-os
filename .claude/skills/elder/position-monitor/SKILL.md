# Position Monitor — 持仓监控

持仓管理是交易中最容易被忽视的环节。
埃尔德的原则：慢慢建仓，快速清仓。止损触发时立即执行，不讨价还价。

---

## 监控内容

对每个持仓，持续追踪：

### 1. 止损状态检查

```
是否触及止损线？
  是 → 立即生成平仓指令，通知用户
  否 → 检查是否需要上移止损
```

**止损上移规则（跟踪止损）：**
- 价格上涨超过 1 倍 ATR → 将止损移至盈亏平衡点
- 价格上涨超过 2 倍 ATR → 将止损移至 1 倍 ATR 盈利处
- 价格上涨超过 3 倍 ATR → 考虑部分止盈，剩余仓位继续持有

止损只能向盈利方向移动，不能反向移动（不能降低止损来"给更多空间"）。

### 2. 趋势健康度检查

通过动力系统（EMA + MACD柱）判断趋势是否还在：

```
动力系统颜色:
  绿色（EMA上升 + MACD柱上升）→ 趋势健康，继续持有
  蓝色（混合信号）→ 趋势减弱，收紧止损
  红色（EMA下降 + MACD柱下降）→ 趋势结束，考虑平仓
```

**注意**：动力系统变红是"禁止加仓"信号，不一定是立即平仓信号。
结合止损位置综合判断。

### 3. 持仓时间检查

- 短期波段交易：持仓超过预期时间（通常5-10天）而没有达到目标，考虑离场
- 信号已经"过期"（原来的入场理由不再成立）→ 离场

### 4. 月度风险状态更新

重新计算每个持仓的当前风险敞口：
```
当前风险 = (当前价格 - 止损价格) × 持仓数量
（如果止损已在盈亏平衡点以上，风险为0）
```

## 紧急情况处理

**止损触发时：**
不要犹豫，不要"再等等看"。
埃尔德原则：止损是你预先设定的退出价格，触发时立即执行。
"触及止损后市场反转"是幸存者偏差——更多时候，不止损会让亏损越来越大。

**市场剧烈波动时（跳空缺口）：**
如果价格跳空越过止损价，以实际成交价平仓，不要等价格回来。

**数据不可用时：**
如果无法获取实时价格，提醒用户手动检查持仓状态，列出需要检查的关键价位。

---

## 与 trade-executor 的协作

position-monitor 发现需要操作的持仓后，生成操作建议传递给 trade-executor：
- 止损触发 → 生成市价平仓指令
- 止损上移 → 生成修改止损单指令
- 部分止盈 → 生成限价卖出指令

---

## 逻辑止损模式（基本面持仓专用）

对于通过 `fundamental-research` 建立的基本面持仓，不使用价格止损。
改为定期检查"逻辑止损条件"（每季度一次，或重大事件发生时）。

**检查清单**（任何一条触发即建议卖出）：
- [ ] 核心竞争优势是否仍然存在
- [ ] 毛利率是否持续下滑（连续 2 季度）
- [ ] 市场份额是否被蚕食
- [ ] 管理层是否有重大变化或诚信问题
- [ ] 财务质量是否恶化（现金流与利润背离加剧）
- [ ] 原始研究报告中的关键假设是否仍然成立

**与价格止损持仓的区分**：
- 持仓备注包含"逻辑止损"字样 → 使用本模式
- 持仓备注包含具体价格止损线 → 使用埃尔德价格止损模式
