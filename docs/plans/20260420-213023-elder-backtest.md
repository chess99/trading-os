## Problem Statement

现有 trading-os 有完整的事件驱动回测引擎（T+1、涨跌停、手数、佣金），但没有 Elder 三重滤网策略。
需要实现 `ElderStrategy`，在历史数据上严格执行 Elder 规则，输出可信的预期收益率。

目标：A 股范围（本地有数据的约 2854 只），只做多方向，2022-01-01 至今。

---

## 核心规则（完整 Elder 方案 B）

### 入场条件（三重滤网全部通过）

**第一滤网（周线，战略方向）：**
- 周线 EMA(26) 方向向上
- 周线 MACD 柱季节为 spring 或 summer（不做 autumn/winter）

**第二滤网（日线，战术时机）：**
- 日线随机指标 K < 30（超卖区）OR 2日强力指数 EMA < 0

**第三滤网（入场执行）：**
- 次日以"前日最高价 + 最小变动单位（0.01元）"挂买入止损单
- 如果次日开盘价 > 止损单触发价，以开盘价成交（跳空处理）

### 仓位计算（2%/6% 原则）

```
每笔最大风险金额 = 账户总值 × 2%
每股风险 = 入场价 - 止损价（= 2 × ATR(13)）
最大仓位 = 每笔最大风险金额 / 每股风险
```

月度累计风险（已亏损 + 持仓敞口）≥ 账户总值 × 6% 时，停止开新仓。

### 止损规则（追踪止损）

- 初始止损：入场价 - 2 × ATR(13)
- 止损只能向上移动，不能下移
- 浮盈达到 1×ATR 时：止损上移至成本线（保护本金）
- 浮盈每增加 1×ATR：止损上移 0.5×ATR

### 卖出规则（A+B 组合）

**规则 A（止损触发）：** 价格跌破追踪止损线，次日开盘卖出

**规则 B（趋势反转）：** 周线动力系统变红（EMA(26) 下降 AND MACD 柱下降），
次日开盘卖出（不管当前盈亏）

两者任一触发即卖出，规则 B 优先级高于规则 A。

### 架构决策：三个规则的实现方式

Reviewer 发现了核心矛盾：追踪止损、条件入场、月度熔断都需要策略层访问它没有的数据。解决方案如下：

**决策 1：入场简化（条件止损单 → 近似为次日开盘买入）**

Elder 原书的"突破前日高点才买入"在当前 BacktestRunner 里无法精确实现（Runner 没有条件单机制）。
**接受近似**：三重滤网通过后，次日以开盘价买入（不检查是否突破前日高点）。
在文档里明确标注这是简化，实际执行价可能略有不同。这对统计结果影响很小。

**决策 2：追踪止损在 generate_signals 里计算**

`generate_signals` 每天都收到最新的 `bars`，可以从中推算当前价格。
追踪止损逻辑放在策略内部：每天调用时，对每个持仓重新计算应该上移的止损线，
如果价格跌破止损线则生成 SELL 信号。

状态（`StrategyContext.extra` 里维护，on_start 初始化一次）：
```python
stops: dict[str, float]         # {symbol: current_stop_price}
entry_prices: dict[str, float]  # {symbol: entry_price}
entry_atrs: dict[str, float]    # {symbol: atr_at_entry}
```

**决策 3：月度熔断用 on_fill 回调追踪**

`Strategy.on_fill(fill)` 在每笔成交后被调用（BacktestRunner 已实现）。
通过 `on_fill` 追踪已实现盈亏，计算月度已用风险额度：

```python
def on_fill(self, fill):
    if fill.action == "SELL":
        pnl = (fill.price - self.entry_prices[fill.symbol]) * fill.shares
        if pnl < 0:
            self.ctx.extra["monthly_loss"] += abs(pnl)
```

未实现的持仓风险敞口（当前止损价 vs 入场价）在 `generate_signals` 里用 `stops` 字典计算。

### MACD 季节定义

| 季节 | MACD 柱值 | MACD 柱方向 | 操作 |
|------|---------|-----------|------|
| spring（春）| < 0 | 向上（今 > 昨）| 最佳做多时机 ✅ |
| summer（夏）| > 0 | 向上 | 做多，持有 ✅ |
| autumn（秋）| > 0 | 向下 | 不做多 ❌ |
| winter（冬）| < 0 | 向下 | 不做多 ❌ |

**第一滤网通过条件：** EMA(26) 向上 AND 季节为 spring 或 summer

### 强力指数公式

```
Force Index(t) = (Close(t) - Close(t-1)) × Volume(t)
2日强力指数 = EMA(Force Index, 2)
```

第二滤网通过条件（满足任一）：
- 日线随机指标 K < 30（超卖）
- 2日强力指数 EMA < 0（多头回调）

### ElderStrategy 设计

```python
class ElderStrategy(Strategy):
    """Elder 三重滤网策略（只做多，完整规则）。
    
    常量（不作为参数，遵循 Elder 原书推荐值）：
        EMA_PERIOD = 26       # 周线 EMA
        ATR_PERIOD = 13       # ATR
        ATR_STOP_MULT = 2.0   # 止损倍数
        STOCH_K = 5           # 随机指标 K 周期
        STOCH_THRESHOLD = 30  # 超卖阈值
        MIN_LOOKBACK_DAYS = 180  # 最少需要 180 天日线（≈ 36 周，保证指标稳定）
    """
    
    def on_start(self, ctx: StrategyContext) -> None:
        self._ctx = ctx          # 存储引用，on_fill 里使用
        ctx.extra.update({
            "stops": {},           # {symbol: stop_price}
            "entry_prices": {},    # {symbol: actual_fill_price（由 on_fill BUY 写入）}
            "monthly_loss": 0.0,
            "current_month": None,
        })
    
    def on_fill(self, fill) -> None:
        """追踪入场价和已实现盈亏，用于追踪止损和 6% 月度熔断。
        
        fill.side: "BUY" | "SELL"（FillEvent 用 side，不是 action）
        """
        extra = self._ctx.extra
        if fill.side == "BUY":
            # 记录实际成交价作为入场价（开盘价近似）
            extra["entry_prices"][fill.symbol] = fill.price
            # 初始止损在 generate_signals 里已写入 stops，这里不重复
        elif fill.side == "SELL" and fill.symbol in extra["entry_prices"]:
            entry = extra["entry_prices"].pop(fill.symbol)
            pnl = (fill.price - entry) * fill.shares
            if pnl < 0:
                extra["monthly_loss"] += abs(pnl)
            extra["stops"].pop(fill.symbol, None)
    
    def generate_signals(self, bars: pd.DataFrame, trading_date: date) -> list[Signal]:
        """
        每个交易日调用。bars 包含所有标的的日线数据（ts < trading_date）。
        
        流程：
        1. 重置月度计数器（如果换月）
        2. 计算 6% 月度熔断状态（已实现亏损 + 未实现止损敞口）
        3. 对每个标的计算周线指标（从日线 resample，使用 W-FRI 边界）
        4. 对已持仓的标的：
           a. 用当前 ATR 更新追踪止损（不用冻结的入场时 ATR）
           b. 检查卖出条件（规则 A：价格 < stop；规则 B：周线变红）
           c. 生成 SELL 信号
        5. 对未持仓且月度熔断未触发的标的：
           a. 检查三重滤网
           b. 计算仓位（2% 原则）
           c. 将初始止损写入 stops 字典（在 BUY 信号生成时，不等 on_fill）
           d. 生成 BUY 信号
        
        追踪止损更新规则（步骤 4a）：
            current_atr = ATR(13) 用最新日线数据计算（不是入场时的冻结值）
            new_stop = max(current_stop, current_price - 2 × current_atr)
            stops[symbol] = new_stop
        
        6% 月度熔断计算（步骤 2）：
            unrealized_risk = sum(
                (entry_prices[sym] - stops[sym]) × position_shares[sym]
                for sym in open_positions
            )
            total_risk = monthly_loss + unrealized_risk
            fuse_triggered = total_risk >= portfolio_value × 0.06
        
        注意：position_shares 从 bars 推断不可靠，需要 BacktestRunner 传入。
        简化：本次实现只用 monthly_loss（已实现亏损）判断熔断，
        不含未实现敞口。这与简化说明表一致，熔断触发稍晚。
        """
```

### 信号格式

给 `Signal` 加 `metadata: dict` 字段（`base.py`，向后兼容）：

```python
@dataclass
class Signal:
    ...
    metadata: dict = field(default_factory=dict)
    # 用于 Elder 策略：
    # metadata = {"entry_price": 6.36, "stop_price": 5.70, "atr": 0.33}
```

`BacktestRunner` 不需要读 metadata（入场近似为开盘价），metadata 只供策略自身的 `on_fill` 回调使用。

### 仓位计算

```python
risk_per_share = entry_price - stop_price  # = 2 × ATR
max_risk_amount = portfolio_value × 0.02   # 2% 原则
max_shares = int(max_risk_amount / risk_per_share / 100) * 100  # 取整到手
size = (max_shares × entry_price) / portfolio_value  # 转为 NAV 比例
```

### 周线 resample 注意事项

使用 `resample('W-FRI')` 而不是 `resample('W')`，避免日历周边界问题。
丢弃最后一个不完整的周（`dropna()`）。
最少需要 36 周（180 交易日）才计算指标，否则跳过该标的。

## 简化说明（与 Elder 原书的差异）

| 规则 | 原书 | 本次实现 | 影响 |
|------|------|---------|------|
| 入场 | 突破前日高点才买入 | 次日开盘价买入 | 入场价略低，胜率可能略高（乐观偏差） |
| 追踪止损 ATR | 每次上移时用当前 ATR | 每次上移时用当前 ATR ✅ | 无简化，完整实现 |
| 仓位 | 按 2%/6% 精确计算 | 按 2%/6% 近似（NAV 比例） | 仓位误差 < 5% |
| 月度熔断 | 含已实现亏损 + 未实现敞口 | 只含已实现亏损 | 熔断触发稍晚，保守方向 |

这些简化不影响策略逻辑的核心验证，但在真实交易中应使用精确版本。
