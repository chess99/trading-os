# ai_quant_trade 深度调研报告

> 仓库：https://github.com/charliedream1/ai_quant_trade
> 调研日期：2026-04-06

## 1. 定位与目标用户

- **核心定位**：一站式量化交易学习与实战平台（~71,796 行 Python）
- **目标用户**：量化研究员、策略开发者、基金经理/操盘手（三类角色）
- **人的角色**：辅助决策为主，AI 生成信号，人工确认后执行

## 2. 系统架构

**技术栈**：Python + backtrader + stable-baselines3 + LangChain + Keras/TensorFlow + Tushare/Wind

**核心目录**：
```
quant_brain/          # 核心量化引擎（回测、数据、资产配置、RL环境）
egs_trade/            # 策略实现（传统/RL/Paper Trade/QLib）
egs_data/             # 数据源适配层（Tushare/Wind/BaoStock/YFinance）
egs_alpha/            # 因子库（Alpha101、TA-Lib、tsfresh 自动提取 5000+ 因子）
egs_llm/              # LLM 应用（热点报告生成、推理型股价预测）
egs_fin_nlp/          # StructBERT 情感分析
egs_aide/             # 辅助工具（Excel 看盘神器）
```

**设计思路**：分层解耦（数据层→信号层→执行层→风控层）+ 策略对象化 + YAML 配置驱动

## 3. AI/Agent 使用方式

**LLM 集成**（egs_llm）：
- 热点报告自动生成：`主题 → TavilySearch → ChatOpenAI → Markdown 报告`
- 推理型股价预测：Unsloth 微调 + vLLM 推理加速

**决策链路**：
```
数据层（OHLCV + 技术指标）
  ├─ 传统规则策略（双均线等）
  ├─ ML/DL 预测（XGBoost/LSTM）
  ├─ LLM 市场分析（热点识别、情感分析）
  └─ RL 策略（PPO/SAC）
        ↓
  信号融合 → 执行决策（带风控规则）
```

**关键 Prompt 结构**（`egs_llm/b01_app/a01_hot_topic_report/`）：
```
###角色### 金融市场分析师
###参考信息### {doc}
###分析主题### {topic}
###任务### 基于参考信息分析
###要求### 忠于原文、简洁但不丢信息
```

## 4. 数据来源与管理

| 数据源 | 市场 | 费用 |
|--------|------|------|
| Tushare | A 股、基金 | 部分付费 |
| Wind | 全球资产 | 机构付费 |
| BaoStock | A 股 | 免费 |
| YFinance | 美股/港股/加密 | 免费 |

**存储**：CSV 本地缓存（`skip_download=True` 避免重复调用）+ 数据格式统一（time/open/high/low/close/volume）

## 5. 策略层

**支持策略**：
- 传统规则：双均线、布林带、RSI、MACD
- 机器学习：SVM、LSTM、随机森林、LGBM
- 强化学习：PPO、SAC、A2C、DDPG、TD3（FinRL NeurIPS2018，A2C 年化 53.1%，夏普 2.17）
- 因子挖掘：tsfresh 自动提取 5000+ 特征 + Alpha101 + TA-Lib

**回测框架**：自建轻量级引擎（account_info + trading_ctrl + risk_indicator）

**风险指标**（ffn 库）：总收益率、年化收益、Beta、夏普比率、最大回撤、索提诺比率

**资产配置**：均匀配置法（`quant_brain/portfolio/capital_allocation.py`）

## 6. 执行层

- **模拟交易**：离线回测历史数据，支持参数扫描
- **实盘模拟**：Wind 本地实盘模拟（机构级，付费）
- **真实交易**：未开源
- **费用结构**（YAML 配置）：卖出税 0.001 + 买入佣金 0.0003 + 最小佣金 5 + 滑点 0.002

## 7. 亮点与可借鉴设计

1. **分层数据抽象** ⭐⭐⭐⭐⭐：`StockDataApi` 抽象基类，多数据源实现同一接口，无缝切换
2. **YAML 配置驱动** ⭐⭐⭐⭐：参数与代码分离，支持命令行覆盖，便于参数扫描
3. **RL 环境标准化** ⭐⭐⭐⭐：遵循 OpenAI Gym 标准，可直接用 stable-baselines3
4. **完整回测可视化**：K 线图 + 买卖点标注 + 资产曲线 + 最大回撤标记
5. **Prompt 工程系统设计**：角色明确 + 结构化输入 + 格式约束 + "忠于原文"要求
6. **工程最佳实践**：`typeguard` 类型检查、`@addlog` 装饰器自动日志、pre-commit 钩子

## 8. 局限性与应避免的设计

| 问题 | 严重程度 | 说明 |
|------|---------|------|
| **RL 观测泄露未来信息** | 严重 ⚠️ | 直接用当天 high/low 作为观测，回测严重高估 |
| 无滑点/停牌/涨跌停处理 | 中等 | 简化模拟导致回测失真 |
| 训练/测试不一致 | 中等 | 测试时加了训练时没学到的人工风控规则 |
| 无多因子融合框架 | 高 | 单一策略，缺信号融合机制 |
| LLM 集成模板化 | 中等 | 仅热点报告，无实时决策链路 |
| 无在线学习框架 | 中等 | 离线训练→固定模型，无增量学习 |

**应避免的做法**：
- 直接用 high/low 作为 RL 观测（必须只用开盘前可知信息）
- 固定参数长期使用（需滑动窗口参数优化）
- 忽视交易成本（精确建模费用+滑点）
- 离线训练→部署（需增量学习应对市场变化）

## 9. 对本系统的启示

**值得采用**：
- 数据抽象层设计（`DataSourceApi` 抽象基类）
- YAML 配置管理 + 命令行覆盖
- Prompt 工程的结构化设计（角色+输入+格式+要求）
- 完整的回测可视化输出

**关键教训**：
- 回测中的信息泄露是最危险的陷阱，必须严格验证
- 数据质量检查是生命线（无穿越信息、数据一致性、缺失值处理、异常值检测）
- 交易成本必须精确建模（费用+税+滑点），否则策略无法实盘
