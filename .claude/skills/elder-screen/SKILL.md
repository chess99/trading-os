# Elder Screen — 三重滤网交易分析

基于《以交易为生》（Alexander Elder）的分析框架，对交易标的进行系统性评估。

## 你需要什么信息

用户提供标的名称（股票代码、期货品种、货币对等）。如果用户没有提供具体价格数据，
使用 web_search 或 browser 工具获取最新行情；如果用户粘贴了数据，直接使用。

---

## 分析框架：三重滤网

埃尔德的核心洞见：**单一时间周期的信号会相互矛盾，必须用多个时间周期过滤**。
趋势跟随指标和震荡指标各有盲点，组合使用才能减少假信号。

### 第一滤网 — 战略方向（长周期）

使用**周线图**确定大趋势方向，这是最重要的一步。

分析要点：
- **EMA 斜率**：26周 EMA 上升 = 牛市，下降 = 熊市，走平 = 中性
- **MACD 柱状线季节**：
  - 春（0值以下向上）→ 最佳做多时机
  - 夏（0值以上向上）→ 持有多仓
  - 秋（0值以上向下）→ 最佳做空时机
  - 冬（0值以下向下）→ 持有空仓
- **结论**：只做多 / 只做空 / 观望

第一滤网是"否决权"：如果周线趋势向下，日线的任何买入信号都忽略。

### 第二滤网 — 战术时机（日线图）

在第一滤网确定方向后，用**日线图**的震荡指标寻找入场时机。

**做多场景**（周线向上）：等待日线出现回调/超卖：
- 随机指标（KDJ）降到下参考线以下
- 强力指数（2日EMA）转负
- 价格回落到两条EMA之间的"价值区间"

**做空场景**（周线向下）：等待日线出现反弹/超买：
- 随机指标升到上参考线以上
- 强力指数（2日EMA）转正
- 价格反弹到价值区间

**背离信号**（最强，优先级最高）：
- 牛市背离：价格创新低，MACD柱底部更高 → 强烈买入
- 熊市背离：价格创新高，MACD柱顶部更低 → 强烈卖出
- 有效背离要求：两个顶/底之间 MACD 柱必须穿越 0 值线

### 第三滤网 — 精确入场

确认前两个滤网一致后，确定具体入场价位：
- **做多**：在前一日最高价上方挂限价买单（价格继续上涨时成交）
- **做空**：在前一日最低价下方挂限价卖单

## 关键指标解读

### 袋鼠尾（Kangaroo Tail）
单根长线柱（高度 ≥ 正常柱的2倍），两侧是普通柱：
- 向上袋鼠尾 → 多方失败，卖出信号
- 向下袋鼠尾 → 空方失败，买入信号
- 止损设在袋鼠尾中点，不是末端

### 支撑/阻力
- 密集交易区持续越久、成交量越大，支撑/阻力越强
- 突破后角色互换（前阻力变支撑）
- 假突破：价格短暂穿越后收回 → 逆方向交易机会

### 成交量确认
- 趋势中成交量萎缩 → 趋势将反转
- 下跌中成交量骤增（恐惧爆发）→ 比上涨中骤增更可能意味着反转

## 资金管理（必须包含在报告中）

**2% 原则**：单笔交易风险不超过账户的 2%

```
最大持仓量 = (账户规模 × 2%) ÷ (入场价 - 止损价)
```

**6% 原则**：本月已亏损 + 所有持仓风险之和 ≤ 账户月初资金的 6%

止损位设置：
- 多头：支撑位下方（避开整数价格，偏移几个最小单位）
- 空头：阻力位上方
- 距离入场价至少 1 倍 ATR（平均真实波幅），避免被噪音触发

## 输出格式

生成一份结构清晰的分析报告，包含以下部分：

```
# [标的名称] 三重滤网分析报告

## 数据时间
[说明数据来源和时间]

## 第一滤网：周线趋势
- EMA 状态：[上升/下降/走平]
- MACD 柱季节：[春/夏/秋/冬]
- 战略方向：[只做多 / 只做空 / 观望]

## 第二滤网：日线入场信号
- 当前位置：[价值区间内/区间上方/区间下方]
- 震荡指标：[随机指标/强力指数状态]
- 背离信号：[有/无，描述]
- 袋鼠尾：[有/无，描述]
- 信号强度：[强/中/弱]

## 第三滤网：入场建议
- 方向：[做多/做空/观望]
- 建议入场价：[价格]
- 止损价：[价格]（理由）
- 第一目标价：[价格]（1倍ATR）
- 第二目标价：[价格]（2倍ATR）
- 风险收益比：[X:1]

## 仓位计算（示例）
假设账户 10万元，2%风险：
- 最大风险金额：2000元
- 每股/手风险：[入场价-止损价]
- 建议仓位：[股数/手数]

## 综合判断
[2-3句话总结：现在是否是好的入场时机，主要风险是什么]

## 需要关注
[列出可能改变判断的关键因素，如即将公布的财报、重要支撑/阻力位等]
```

---

## 注意事项

- 如果第一滤网和第二滤网方向冲突，**明确说明不建议入场**，等待方向一致
- 数据有限时，诚实说明哪些指标无法计算，基于现有信息给出最佳判断
- 这是技术分析，不是基本面分析——不对公司价值做判断
- 始终提醒用户：这是参考意见，不是投资建议，交易有风险

## Trading OS 集成

当用户提供股票代码时，优先从本地数据湖获取数据：

```bash
# 先确认本地有数据
python -m trading_os query-bars --symbols SSE:600000 --adjustment qfq --limit 5

# 如果没有，先获取
python -m trading_os fetch-ak --exchange SSE --ticker 600000 --start 2022-01-01 --adjustment qfq
```

获取到 K 线数据后，用数据中的 close/open/high/low/volume 计算各指标，
不要依赖 web_search 获取价格（避免数据不一致）。

分析完成后，如果产生了交易信号，可以用 `trading-os agent` 命令生成结构化信号，
或直接调用 `position-sizer` skill 计算仓位。
